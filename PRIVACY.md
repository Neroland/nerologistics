# NeroLogistics privacy & telemetry disclosure

NeroLogistics includes optional, anonymous error reporting so that crashes and bugs caused by the
mod can be found and fixed. This page is the full disclosure required by CurseForge's moderation
rules for mods that use an external analytics/error service, and it documents how the mod is
designed to comply with the GDPR (EU) and POPIA (South Africa).

> **Note:** error reporting only becomes active once NeroLogistics' own Sentry DSN is configured in
> the build (`NeroLogisticsTelemetry.DSN`). While that field is blank, NeroLogistics sends
> **nothing**, regardless of the settings below.

## What is collected (error reporting)

When an error **caused by NeroLogistics code** occurs, a report is sent to
[Sentry](https://sentry.io) (hosted in the **EU** — `ingest.de.sentry.io`) containing:

- the stack trace of the error and the error message (file paths containing your operating-system
  account name are masked before sending),
- the NeroLogistics version, the loader (NeoForge / Forge / Fabric), and whether the game is a
  client or dedicated server,
- the operating system name/version and Java version,
- one of the mod's own configuration flags (whether per-player throughput attribution is enabled),
  so a crash that only happens at non-default settings can be reproduced,
- a random event ID generated by Sentry for the individual report.

Reports are de-duplicated and hard-capped at 10 events per game session. **No names, UUIDs, IP
addresses, chat, coordinates, or world data are ever sent.**

## How to opt out

Set `telemetryEnabled=false` in `config/nerologistics.properties` (client-local; takes effect on
restart). Error reporting then sends nothing.

## Player data inside the mod (POPIA / GDPR)

NeroLogistics stores **no personal data by default** — networks, terminals, ports, drones and
dashboards are keyed by block position and dimension, never by player.

**Per-player throughput attribution is opt-in and off by default**
(`perPlayerThroughputAttribution`). When an operator turns it on, the mod records only the **UUID**
of the player who placed a cargo port (never a name), is **retention-pruned**
(`attributionRetentionDays`, default 30 days), and is purged through **Neroland Core's shared
data-erasure hook** — both on an explicit erase request and by Core's inactivity sweep. The opt-in
flag is also the server toggle to disable personal-data logging entirely.

See the [wiki Dashboard & Privacy page](wiki/Dashboard-and-Privacy.md) for the in-game side.
